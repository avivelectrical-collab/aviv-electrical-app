## 🏗️ ארכיטקטורה של הפרויקט

### Stack
- **Frontend:** React (כרגע הכל בקובץ `index.html` אחד)
- **Backend/DB:** Firebase (Firestore + Auth)
- **אחסון:** Firebase Storage
- **פריסה:** Vercel (אוטומטי מ-GitHub main branch)
- **URL חי:** https://aviv-electrical-app.vercel.app
- **GitHub:** https://github.com/avivelectrical-collab/aviv-electrical-app

### מבנה נוכחי
- `index.html` — קובץ ענק (~1MB) שמכיל את כל הקוד. זה **חוב טכני** שנרצה לטפל בו בעתיד אבל רק כשנחליט על זה במפורש.
- כל שינוי נעשה בקובץ הזה עד שנחליט לפצל.

### רכיבים עיקריים (מודולים לוגיים בתוך index.html)
1. **🟦 פרויקטים** — רשימה + tabs לפי סטטוס (פעילים/מושהים/הושלמו/הכל)
2. **🟦 יומן עבודה** — סטטוס לכל סעיף (✅/🔄/⏳), שעות, עובדים, פאנל משימות פתוחות
3. **🟦 חשבונות** — חשבונות חלקיים/סופיים, קיזוזים, עיכבון, סטטוסים, ולידציות תאריכים
4. **🟦 הוצאות נלוות** — checkbox "לקיזוז", עמודות קוזז/יתרה
5. **🟦 תמחור BOQ** — השוואה למחירון דקל, פאנל פירוט חומרים, כפתור שימוש במחיר דקל
6. **🟦 תזרים מזומנים** — טבלה חודשית, הכנסות צפויות אוטומטיות, הוראות קבע, ספריית ספקים, תזרים יומי
7. **🟦 זיהוי PDF של ספקים** — AI מזהה → אישור → שמירה בתזרים + עדכון מחירון

## 📝 Git / Commits

### הודעות commit בעברית
דוגמאות לפורמט:
```
הוסף: טאב "משימות פתוחות" ליומן עבודה
תיקון: באג בוולידציית תאריכים בחשבונות
שיפור: ביצועי טעינת טבלת תזרים מזומנים
רפקטור: פיצול רכיב BOQPricing
```

### לפני commit
1. בדוק שה-build עובר
2. בדוק שלא שברת פיצ'ר קיים (בדיקה ידנית של המודולים הקשורים)
3. תן סיכום קצר של מה השתנה

### Push
- push ל-main מפעיל build אוטומטי ב-Vercel
- **שאל אותי לפני push** אם השינוי משמעותי

## 🎯 איך אני אוהב שתעבוד

### כשאני מבקש פיצ'ר חדש
1. **שאל** — מה בדיוק? איפה בממשק? אילו מקרי קצה?
2. **תכנן** — הצג לי תוכנית קצרה לפני שאתה נוגע בקוד
3. **תעבוד בצעדים קטנים** — שינוי אחד, בדיקה, שינוי הבא
4. **תראה לי** — הראה מה השתנה לפני commit

### כשאני מבקש תיקון באג
1. **תבין את הבעיה** — קרא את הקוד הרלוונטי, לא תנחש
2. **תסביר לי את הסיבה** — לא רק "תקנתי"
3. **תציע פתרון** — ותגיד לי אם יש trade-offs

### כשאתה נתקל בבעיה
- אל תמציא תשובות. אם אתה לא בטוח, תגיד.
- אם חסר לך מידע, תשאל.
- אם המשימה מסובכת מדי, תגיד ותציע לחלק אותה.
